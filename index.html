Hacktoberfest: Celebrate Open Source

A friendly guide to participating in Hacktoberfest — learn how to find projects, submit PRs, and earn community swag.

Who can join?

Anyone — students, beginners, and experienced devs. Contributions can be code, docs, translations, or issues (depends on program rules).

What counts?

Valid pull requests to participating repositories. Always read the repo's contribution guidelines and maintainers' labels (e.g., good first issue).

Pro tip

Look for repos with labels like good first issue, beginner, or help wanted.

How to participate Read FAQ

Quick checklist

  • Create or sign in to a GitHub account
  • Find an open-source project to contribute to
  • Open a PR following repo rules
  • Repeat until you meet the event's PR requirement
  • Enjoy swag / recognition (if provided)

About Hacktoberfest

Hacktoberfest is a month-long event that encourages contributions to open source. It’s a great time to start contributing, learn collaboration workflows, and help maintainers.

How to participate — step by step

1
Register or sign in
Sign in to GitHub. Some program editions may require explicit registration on the official site — check the official event page for the current year.
2
Find issues to work on
Search repositories for good first issue, help wanted, or language-specific tags. Read CONTRIBUTING.md and open an issue or comment to ask before starting where appropriate.
3
Make your changes
Fork the repo, create a branch, implement your fix, add tests or docs if needed, and push your branch.
4
Open a pull request
Open a PR with a clear title and description. Reference any issues and follow template guidelines. Be patient and responsive to maintainer feedback.
5
Repeat
Submit the number of valid PRs required by that year's rules (often 4) to earn event eligibility. Rules and validity checks vary by organizer.

Common timeline / tips

Hacktoberfest usually takes place in October. Repos and maintainers may set rules and label requirements — always check current event guidance before assuming anything.

Before October

Find projects, read the contributing guide, and discuss issues with maintainers.

During October

Open PRs, respond to reviews, and help others. Keep PRs meaningful — avoid spammy or trivial changes unless asked.

After the event

Follow up on merged PRs, keep contributing, and thank maintainers — healthy open-source communities need sustained help.

FAQ

Is Hacktoberfest free?
+
Participation is free. Some editions may offer paid swag or prizes — check the official event page for the year's specifics.
Do documentation PRs count?
+
Often yes. Many maintainers welcome documentation, translations, tests, and other non-code contributions. Confirm by reading repository contributing guidelines.
What makes a PR invalid?
+
Invalid PRs may include spam, trivial whitespace-only changes, or changes made without following repo rules. Always follow maintainers' guidance.
